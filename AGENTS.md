# AGENTS.md — Gemma Data Assistant

## Project Overview
Streamlit app that lets users upload CSV/Excel files and chat with a local Gemma 4 E4B model via llama.cpp's `llama-server` (OpenAI-compatible API on port 8080).

## Startup Commands
```bash
# 1. Start llama-server optimized for RTX 4060 8GB (128K context, flash attn, KV quantization)
./llama-opencode.bat

# 2. Run the app
.venv\Scripts\Activate.ps1
streamlit run app.py
# Or just: run_app.bat (auto-creates venv + installs deps)
```

## Architecture
- **Single file app**: `app.py` is the entire application (~1200 lines)
- **Server**: `llama-server` runs externally on `http://localhost:8080/v1` — the app connects via `openai` Python SDK
- **Model**: Gemma 4 E4B GGUF from HuggingFace cache (`%USERPROFILE%\.cache\huggingface\hub\models--unsloth--gemma-4-e4b-it-gguf\`)
- **Multimodal projector**: `mmproj-BF16.gguf` in same cache dir — required for audio/image inputs

## Key Dependencies
- `streamlit` — UI framework
- `pandas` + `openpyxl` — data loading (CSV/XLSX)
- `openai` — client for llama-server's OpenAI-compatible API
- `matplotlib` + `seaborn` + `openpyxl` — chart generation and Excel report creation (executed via AI tool calls)
- `tabulate` — DataFrame to Markdown conversion

## Critical Implementation Details

### Data Context Pipeline
1. File uploaded → Pandas reads it → converts object columns to strings
2. Schema + pre-calculated statistics (describe + sum) injected into system prompt
3. ⚡ **Fast Context** toggle (default ON): sends only top 100 rows instead of 2000 to speed up inference
4. AI receives full stats table for accurate math without reading all rows

### AI Tool System
- Single tool: `execute_python_code` — AI generates Python code, app executes it via `exec()`
- Code runs with `df`, `pd`, `plt`, `sns`, `openpyxl`, `datetime`, `numpy`, `dataframe_to_rows`, `Image`, `json`, `re`, `math` pre-loaded
- **Sandboxed**: `__import__` blocked — no import statements allowed. All needed modules are pre-imported.
- Results (charts as base64, code, dataframes, Excel files) are **persisted in session state** so they survive reruns
- Tool results are stored in `message["tool_results"]` and re-rendered in chat history
- Excel reports: AI creates openpyxl workbook → saves to BytesIO → app detects xlsx ZIP magic bytes and offers download
- Multiple figures saved individually (each figure encoded as separate base64)

### Context Window Management
- Token counting before each API call (estimate: `len(json.dumps(msg)) // 4`)
- Auto-trims oldest user/assistant pairs when exceeding 96K tokens (safe under 128K limit)
- Sidebar shows combined data + conversation token usage with overflow warning

### Voice Input
- Uses `st.audio_input()` → sends WAV bytes directly to llama-server's `/v1/audio/transcriptions` endpoint
- ✅ **Fully offline** — Gemma 4 E4B native ASR (Conformer audio encoder via mmproj-BF16.gguf)
- MD5 hash prevents duplicate processing of same audio blob
- Transcribed text replaces the prompt sent to the AI
- Requires `--mmproj` flag on llama-server (included in `llama-opencode.bat`)

### Thinking Mode
- `<|think|>` token injected at start of system prompt
- App parses `<think>`/`</think>` tags to separate reasoning from final answer
- Only final answer is saved to chat history (per Gemma 4 spec)

### Theme
- **Dark mode only** — hardcoded DeepMind Navy palette in `c` dict
- No theme toggle — removed to avoid light mode contrast bugs

## Gotchas
- **llama-server must be running** before the app can respond to queries
- **Audio requires `--mmproj`** flag on llama-server — without it, audio/image inputs return 500
- **Charts disappear on rerun** if not persisted in `tool_results` — always save plot as base64
- **Session state resets** when new file uploaded — chat history is cleared intentionally
- **Streamlit reruns on every interaction** — use session state for all persistence
- **`st.audio_input` persists across reruns** — always use MD5 hash to detect new recordings
- **Code sandbox blocks `exec`/`eval`** in AI code — the AI cannot call exec() inside its generated code
- **Context window auto-trims at 96K tokens** — oldest message pairs dropped to stay under limit
- **Excel detection scans for ZIP magic bytes** — any BytesIO with `PK` header is treated as xlsx
